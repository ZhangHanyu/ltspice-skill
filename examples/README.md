# LTspice Skill Examples

Each subdirectory contains a self-contained LTspice example demonstrating a different simulation type.

## Structure

Source files (checked in):
- `*.asc` — LTspice schematic (primary source, edit this)
- `*.net` — Netlist (auto-generated by LTspice from `.asc`, or hand-written when no `.asc` exists)

Generated files land in `output/` (gitignored; re-running the example overwrites them):
- `output/*.raw` — Binary waveform data
- `output/*.op.raw` — Operating point data (if present)
- `output/*.log` — Simulation log
- `output/*.csv` — Converted waveform data

---

## TRAN_analysis — Transient Analysis (RC Filter)

Simple RC low-pass filter. Demonstrates basic `.TRAN` simulation and CSV export.

```powershell
# Run from examples/TRAN_analysis/ directory
LTspice.exe -run -b .\RC_filter.asc
Move-Item RC_filter.raw, RC_filter.op.raw, RC_filter.log .\output\
..\..\converter\ltspice_raw2csv.exe .\output\RC_filter.raw -d
..\..\converter\ltspice_raw2csv.exe .\output\RC_filter.raw -o .\output\RC_filter.csv --traces "time,V(vin),V(vout)" -q -f
```

## AC_analysis — AC Analysis (RLC Lowpass Filter)

2nd-order passive RLC lowpass filter (Fc ≈ 15.9 kHz). Demonstrates `.AC` simulation and complex number export modes (`ri` and `ma`).

```powershell
# Run from examples/AC_analysis/ directory
LTspice.exe -run -b .\RLC_lowpass.net
Move-Item RLC_lowpass.raw, RLC_lowpass.op.raw, RLC_lowpass.log .\output\

# Real/imaginary columns (default)
..\..\converter\ltspice_raw2csv.exe .\output\RLC_lowpass.raw -o .\output\RLC_lowpass_ri.csv --traces "frequency,V(out)" -q -f

# Magnitude/angle columns (Bode-plot ready)
..\..\converter\ltspice_raw2csv.exe .\output\RLC_lowpass.raw -o .\output\RLC_lowpass_ma.csv --traces "frequency,V(out)" --complex-mode ma -q -f
```

## STEP_analysis — Parameter Sweep (.STEP)

RC low-pass filter with a `.STEP` sweep over resistance (R = 1 kΩ, 2 kΩ, 3 kΩ). Demonstrates how all steps are exported with `step` and parameter columns, and how `--step N` selects a single run.

Note: the `.log` file must be present alongside `.raw` for step boundaries and parameter values to be read.

```powershell
# Run from examples/STEP_analysis/ directory
LTspice.exe -run -b .\RC_step.net
Move-Item RC_step.raw, RC_step.op.raw, RC_step.log .\output\

# All steps — prepends 'step' and 'r' columns
..\..\converter\ltspice_raw2csv.exe .\output\RC_step.raw -d
..\..\converter\ltspice_raw2csv.exe .\output\RC_step.raw -o .\output\RC_step_all.csv --traces "time,V(vout)" -q -f

# Single step (step 2: R=2k)
..\..\converter\ltspice_raw2csv.exe .\output\RC_step.raw -o .\output\RC_step2.csv --traces "time,V(vout)" --step 2 -q -f
```

## FRA — Frequency Response Analysis (Op-Amp Loop Gain)

Inverting op-amp amplifier (gain = −10, GBW = 10 MHz) with FRA loop gain measurement. Demonstrates `.FRA` simulation with `@` FRA device and complex Bode-plot output.

Note: FRA produces `opamp_fra.fra_1.raw` (not `opamp_fra.raw`). The background transient `.raw` is large and should be deleted after running.

```powershell
# Run from examples/FRA/ directory
LTspice.exe -run -b .\opamp_fra.net
Move-Item opamp_fra.fra_1.raw, opamp_fra.op.raw, opamp_fra.log .\output\
# Delete large background transient
Remove-Item opamp_fra.raw -ErrorAction SilentlyContinue

# Convert FRA output (use --complex-mode ma for Bode plot)
..\..\converter\ltspice_raw2csv.exe .\output\opamp_fra.fra_1.raw -o .\output\opamp_fra_bode.csv --complex-mode ma -q -f
```
